7.4. Типы свойств ресурса

Тип свойства

характеристики свойства ресурса , которые определяют правила заполнения и функциональные особенности.

Каждое свойство ресурса может иметь один из следующих типов:

Comment

Значением свойства является произвольная строка. Чаще всего значение свойств типа Comment вычисляются при помощи функций. Значения свойств типа Comment формируют название проключения.

Config Point

Значением свойства является точка конфигурации подключения, выбираемая из списка возможных для заданного ресурса центров конфигурации.

Hidden Config Point

Значением свойства является точка конфигурации проключения, найденная в родительских ресурсах, имеющих заданные центры конфигурации.

Hidden

Значением свойства является произвольная строка. Чаще всего значение свойств типа Hidden вычисляются при помощи функций.

IP-address

Значением свойства является 

x.x.x.x (ip адрес)
Restricted Rights

Используется только для ресурсов, выбранных, как Доступ к серверу статистики. Принимает два значения: истина или ложь. В случае, если значение свойства Истина, пользователь стат-сервера имеет доступ к системе в режиме Только чтение.

Popup

Тип свойства подобный типу Авториз.URL, но открывающий ссылку в отдельном маленьком окне.

SNMP Oid

SNMP Oid.

URL

Просто URL.

Авториз. URL

URL с аутентификацией.

Логин

Логический

Метод шифрования пароля

Метод шифрования пароля

Многомерный объект

Номер заказа

Номер из серии

Значением свойства является номер из серии номеров.

Пароль

Значением свойства является криптованная строка символов. Метод криптования определяется свойством типа "Метод шифрования пароля". Если свойство типа Метод шифрования не включено в ресурс, используется алгоритм криптования DES.

Подсеть

Значением свойства является строка типа (ip-подсеть).

x.x.x.x/x

Пример 7.2. Применение типа свойства «Подсеть»

Предположим, что в системе есть ресурс A одним из свойств A1 которого является Подсеть. При подключении ресурса A система автоматически устанавливает один фингерпринт подключения подсети и статистика потребления трафика считается по единственному фингерпринту, т.е. на всю подсеть. Чтобы автоматически считать статистику на каждый IP-адрес подсети вводится функция Свойство (псевдо) для специального свойства А2 ресурса A. Свойство A2 должно иметь тип Список свойств для разбиения. В качестве значения по умолчанию свойства А2 укажем функцию Свойство (псевдо). В списке параметров функций для свойств создаем функцию для A2, указываем, какое именно свойство необходимо разбить. В нашем случае это свойство A1. В системе устанавливается связь между двумя свойствами. При подключении ресурса A, оператор может указать в качестве значения свойства A2 свойство A1. В этом случае для каждого IP-адреса подсети (значение свойства A1) будет сформирован фингерпринт и статистика потребления трафика будет автоматически собираться по каждому IP-адресу. В данном случае мы использовали функцию Свойство (псевдо) для связи свойства типа Подсеть со свойством типа Список свойств для разбиения с целью детализации статистики по каждому адресу подсети. Такая же конструкция справедлива для связи свойств типа Серия номеров и Список свойств для разбиения с целью детализации статистики по каждому номеру серии.


Профиль пользователя

Используется для сервера статистики и регистрации. В системе можно определить несколько профилей доступа к данным.

Серийный номер

Серийный номер подключения

Серия номеров

Значением свойства является строка вида N/M. N - первый номер серии. M - количество номеров в серии. Приведем пример использования свойства этого типа.

Пример 7.3. Применение типа свойства «Серия номеров»

Предположим, что в системе есть ресурс A, одним из свойств A1 которого является серия номеров - 1/50. Мы же осуществляем подключение ресурса B, которому нужно выделить только несколько последовательных номеров из этой серии, например пять последовательных номеров из серии A1.

Первое, что требуется сделать - это связать ресурс B с ресурсом A. Затем, чтобы описать процесс выделения серии, вводится функция Свойство (псевдо) для свойства B1 ресурса B . Свойство B1 должно иметь тип Серия номеров (чтобы выделение серии имело смысл), значением по умолчанию свойства B1 делается функция Свойство (псевдо). В списке параметров функций для свойств создаем эту функцию для нашего свойств B1: указываем точку входа в ресурс (в данном случае в ресурс A ), это, как правило свойство AN, образующее список, например - AN = Название. Автоматически среди свойств ресурса A будет найдено свойство того же типа ( Серия номеров), как свойство B1 - в нашем случае A1, и между ними будет установлена связь. Оператор может попросить систему показать оставшиеся свободными у A1 серии номеров, выбрать и внести серию в значение свойства B1.

Например, в серии 1/50 заняты 1/15, 18/5, 40/5. Тогда можно взять первые пять последовательных номеров - 23/5. В данном примере мы использовали функцию Свойство(псевдо) для связи свойства типа Серия номеров одного ресурса B с свойством типа Серия номеров другого ресурса A с целью автоматического контроля за операторским выделением серий номеров. Такая же конструкция справедлива для связи свойств типа Подсеть с целью выделения последовательного пула IP-адресов из подсети.

Значением свойства является символьная строка
